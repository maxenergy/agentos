# understand-system

## Purpose

Build a mental model of an existing codebase before making changes.

## Steps

1. List project structure.
2. Identify entrypoints.
3. Identify core modules.
4. Trace data flow.
5. Identify high-risk files.
6. Identify "do not touch" areas.

## Output

Write to:

- docs/system/ARCHITECTURE.md
- docs/system/MODULE_MAP.md
- docs/system/DATA_FLOW.md
- docs/system/RISK_AREAS.md

## Rules

- Do not change code.
- Do not guess architecture.
- If unclear, mark as unknown.
